# hotel_neo4j
Creación base de datos Neo4j y consultas con equialente en sql


En el presente repositorio se describe de forma general
el procedimiento de transformacion de una base de datos
relacional a una base de datos no relacional basada en Grafos.
El motor usado es Neo4J el cual tiene una estructura flexible
definida por las relaciones almacenadas entre registros de datos.
Las consultas y operaciones entre las relaciones se ejecutan a
trav´es de Cypher que es un lenguaje optimizado para realizar
estas tareas. Adicionalmente, este articulo describe el proceso
de instalaci´on tanto del motor Neo4J como de Cypher, algunos
ejemplos de consultas hechas sobre una base de datos ejemplo
en SQL y la discusi´on de resultados expuesta al finalizar el
documento donde se exponen algunos apartados del proceso de
transformacion.
Index Terms—NoSQL, SQL, consultas, Cypher, Neo4J
